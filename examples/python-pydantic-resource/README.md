# Pydantic Example Resource

This example demonstrates how to write a DSC resource using [Pydantic v2](https://docs.pydantic.dev/) for schema definition and runtime validation.

## When to use Pydantic

Pydantic is optional in the Python adapter. Use it when your resource:
- Needs **runtime validation** of input properties
- Has **complex type constraints** (ranges, patterns, custom validators)
- Already depends on Pydantic for other purposes
- Requires **better error messages** for invalid inputs

For simple resources with no validation requirements, the built-in `DataclassSchemaProvider` (zero dependencies) is recommended.

## Key differences from dataclass approach

| Aspect | Dataclass | Pydantic |
|--------|-----------|----------|
| **Base class** | `@dataclass` decorator | `BaseModel` |
| **Dependencies** | None (stdlib) | `pydantic>=2.0` |
| **Schema provider** | `DataclassSchemaProvider` | `PydanticSchemaProvider` |
| **Validation** | None | Automatic at construction |
| **Error handling** | Silent (unknown fields ignored) | Raises `ValidationError` |
| **Type hints** | Python types | Pydantic-enhanced types |
| **Field defaults** | `field(default=...)` | `Field(default=...)` |

## Resource layout

```
my_package/
├── __init__.py
├── my_resource.py           # Resource implementation
├── dsc/                      # Generated by dsc-gen
│   └── *.dsc.adaptedResource.json
pyproject.toml              # Entry points + dependencies
```

## Installation & setup

```bash
# Install package in development mode
pip install -e .

# Generate adapted resource manifests (optional; done at wheel-build time)
dsc-gen manifest --out my_package/dsc/
```

## Resource capabilities

This example implements:
- **Get** — Read current state
- **Set** — Apply desired state (returns state + changed properties)
- **Test** — Compare actual vs desired (returns state + diffs)
- **Delete** — Remove the managed instance
- **Export** — Enumerate all instances

## Validation features

The Pydantic model includes:

```python
class MySchema(BaseModel):
    name: str = Field(min_length=1, max_length=255)  # Constraints
    _exist: bool = Field(default=True)

    @field_validator("name")
    @classmethod
    def validate_name(cls, v: str) -> str:
        if v.lower() == "reserved":
            raise ValueError("name cannot be 'reserved'")
        return v
```

This means:
- `name` must be 1–255 characters (enforced at validation time)
- `name` cannot be "reserved" (custom validator)
- If DSC passes invalid data, Pydantic raises `ValidationError` with clear messages
